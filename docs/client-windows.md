# Windows 客戶端設定

本文件說明如何在 Windows 上安裝與設定 RustDesk 客戶端，連線至自架 RustDesk Server（社群版）。

> **使用前須知**：連線伺服器所需的 **ID 伺服器地址**、**API 伺服器地址** 與 **Key（公鑰）**
> 為實驗室內部資訊，請向**實驗室 IT 管理員**申請取得後再進行設定。

> 其他桌面平台（macOS、Ubuntu、Flatpak）的設定流程與本文一致，僅安裝方式不同。

詳細操作請參考 [RustDesk 官方文件](https://rustdesk.com/docs/zh-tw/client/)。

## 安裝

### 方式一：安裝版（推薦）

1. 前往 [RustDesk GitHub Releases](https://github.com/rustdesk/rustdesk/releases/tag/1.4.6) 下載 v1.4.6 或最新版之 Windows 安裝檔（例如 `rustdesk-1.4.6-x86_64.exe`）。
2. **⚠️ 隱私功能支援警告**：在下載好的安裝程式（`.exe`）上點擊**右鍵**，並選擇**「以系統管理員身分執行」**來進行安裝。若未以系統管理員權限安裝，日後將無法正常使用「隱私模式 (Privacy Mode)」（如黑屏防窺）等進階被控端功能。
3. 依畫面指示完成安裝。
4. 安裝後 RustDesk 會自動以系統服務模式運行，支援無人值守存取。

### 方式二：免安裝（Portable）

直接執行下載的 `.exe` 即可使用，但：

- 不會安裝系統服務，需手動啟動
- 無法作為被控端接受無人值守連線
- 適合臨時使用場景

## 設定自架伺服器

安裝完成後，需將客戶端指向你的自架伺服器（預設指向 RustDesk 公共伺服器）。

### 步驟

1. 開啟 RustDesk
2. 點擊主視窗左下角的 **選單圖示**（三條橫線）
3. 選擇 **網路 → ID/中繼伺服器**
4. 填入以下資訊（向管理員索取）：

| 欄位 | 值 | 說明 |
|------|---|------|
| **ID 伺服器** | `rustdesk-server.example.com` | 伺服器域名或公網 IP（直連） |
| **中繼伺服器** | `rustdesk-server.example.com` | 同主機可留空，自動使用 ID 伺服器 |
| **API 伺服器** | `https://rustdesk-console.example.com` | API（走 Cloudflare Tunnel） |
| **Key** | `<id_ed25519.pub 的內容>` | 向管理員索取 |

> **注意**：
> - **ID/中繼伺服器**只填域名或 IP，不加 `http://` 或 port
> - **API 伺服器**必須填完整 HTTPS URL（含 `https://`）
> - API 伺服器是 Pro 版功能，提供地址簿同步、裝置管理等

5. 點擊 **確定** 儲存

### 驗證

設定完成後，主視窗下方狀態列應顯示 **「就緒」**。若顯示「尚未就緒」或持續嘗試連線，請檢查：

- ID 伺服器域名 / IP 是否正確
- Key 是否完整貼上（不可有多餘空格或換行）
- 伺服器防火牆 / 路由器是否開通 21116/udp
- API 伺服器是否填入了正確的 HTTPS URL（走 Cloudflare Tunnel 的域名）

## 連線遠端設備

### 作為控制端（連線他人）

1. 在主視窗的「控制遠端桌面」欄位輸入**對方的 ID**
2. 點擊 **連接**
3. 輸入對方設定的連線密碼
4. 進入遠端桌面

### 作為被控端（讓他人連線）

1. 主視窗左側顯示你的 **ID** 和**一次性密碼**
2. 將 ID 和密碼告知控制端使用者
3. 如需無人值守，設定固定密碼：
   - 點擊密碼旁的 **編輯圖示**
   - 選擇 **設定固定密碼**
   - 輸入強密碼並儲存

## 常用功能

### 檔案傳輸

連線後點擊工具列的 **檔案傳輸** 圖示，即可在本機與遠端之間互傳檔案。

### 剪貼簿同步

預設啟用，本機與遠端的複製貼上會自動同步。

### 多螢幕切換

遠端若有多個螢幕，工具列會顯示螢幕切換按鈕。

### 調整畫質

工具列 → **顯示** → 可切換「最佳畫質」或「流暢優先」。

## 常見問題

### 連線時提示「Key 不符」

客戶端填入的 Key 與伺服器的公鑰不一致。請向管理員重新索取 `id_ed25519.pub` 的內容。

### 狀態列長時間顯示「連接中...」

1. 確認 ID 伺服器欄位格式正確（僅填域名或 IP，不加 `http://` 或 port）
2. 確認本機網路可達伺服器的 21116 port
3. 確認 API 伺服器欄位為完整 HTTPS URL（`https://...`）
4. 嘗試在命令提示字元測試：
   ```
   ping rustdesk-server.example.com
   ```

### 無法進行無人值守存取

確認已使用**安裝版**（非 Portable），且 RustDesk 服務正在運行：

1. 按 `Win + R` → 輸入 `services.msc` → Enter
2. 找到 **RustDesk Service**，確認狀態為「執行中」

### 被防火牆 / 防毒軟體攔截

RustDesk 需要網路存取權限。將 `rustdesk.exe` 加入防火牆與防毒軟體的允許清單。
